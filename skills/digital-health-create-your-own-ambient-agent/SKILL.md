# Digital Health Create Your Own Ambient Healthcare Agent

Build a complete ambient healthcare voice-agent repository through a
spec-driven development loop. The architecture is fixed; the developer customizes
clinical workflows, tools, data sources, graph pattern, models, and guardrails.

Do not use this skill for general chatbots, non-healthcare voice demos, or
standalone LangGraph/LangChain projects that do not need the full ambient voice
agent repository structure.

## Instructions

Follow the phases in order. Each phase has a dedicated reference file; read the
referenced file before doing that phase. Do not skip approval gates.
When a reference marks developer-facing text as **verbatim**, send that text
exactly as written instead of summarizing or paraphrasing it.

1. **Prerequisite references**: Read
   `references/prerequisite-references.md`. Clone or verify the reference repos,
   verify the required dependent sub-skill `SKILL.md` files are readable, and
   optionally symlink sub-skills into the active coding-agent skill directory.
2. **Specify**: Read `references/specification-workflow.md`. Ask the required
   planning questions and write `specs/<feature>/spec.md` files. Present each
   spec and wait for explicit developer approval before planning.
3. **Plan**: Read `references/planning-workflow.md`. Derive
   `specs/<feature>/plan.md` and `tasks.md` from approved specs. Present every
   plan and task list and wait for approval before implementation.
4. **Implement**: Read `references/implementation-guide.md`. Generate the
   output repository from approved tasks, use the required reference
   implementations and sub-skills, and keep `agent/` and `config/` mounted at
   Docker runtime instead of baked into the image.
5. **Validate and hand off**: Read `references/validation-handoff.md`. Run the
   required static checks, unit tests, FastAPI integration tests, optional
   guardrails tests, Docker Compose validation, and final handoff.

If the developer asks about the generated repo's README text or handoff wording,
read `references/output-repo-template.md` and `references/validation-handoff.md`.
If the developer asks about detailed design constraints or Phase 2 tradeoffs,
read `references/design-requirements-guide.md`.
If the developer asks for a visual overview of the skill workflow, read
`references/workflow-flowchart.md`.

## Fixed Architecture

```text
[Browser mic]
      |  WebRTC audio
      v
[Nemotron Voice Agent]   NVIDIA Blueprint: pipecat, ASR, TTS
      |  POST /v1/chat/completions  (OpenAI-compatible)
      v
[Agent Backend]          FastAPI server generated by this skill
      |  LangGraph agent
      v
[Tools]                  developer-defined functions
      |  DB / API / web calls
      v
[Data Layer]             developer-chosen backend
```

- **Frontend**: NVIDIA Nemotron Voice Agent with WebRTC, pipecat, Riva ASR/TTS.
- **Server**: FastAPI exposing `/v1/chat/completions`; the voice agent calls this
  endpoint instead of a raw LLM NIM.
- **Agent**: LangGraph; the graph pattern and tools are selected by the developer.
- **Runtime**: Docker Compose remains the primary launch path.

Reference implementations live in `references/langgraph-backend/`. Read them
before writing code:

- `references/langgraph-backend/graph.py`: LangGraph ReAct pattern and streaming
  filter reference.
- `references/langgraph-backend/server.py`: FastAPI OpenAI-compatible server.
  Copy and adapt it; do not rewrite the SSE logic from scratch.
- `references/langgraph-backend/tools/example_tool.py`: tool template.
- `references/langgraph-backend/requirements.txt`: minimal dependency set.

## Required Gates

- **No Phase 2 before prerequisites**: the three reference repos and five
  dependent sub-skill files must be readable.
- **No planning before spec approval**: every selected feature needs an approved
  `spec.md`.
- **No implementation before plan approval**: every selected feature needs an
  approved `plan.md` and `tasks.md`.
- **No handoff before validation**: static checks, tests, Docker Compose checks,
  and generated runnable-skill audit must pass or be reported as blockers.

Selected features are always:

- `langgraph-agent`
- `fastapi-server`

Add `guardrails` only if the developer enables NeMo Guardrails during Phase 2.

## Installation Notes

Install this entire folder, not just `SKILL.md`; the skill depends on
`references/`, `scripts/`, and `evals/`.

For local development from the skill directory, set:

```bash
SKILL_SRC="$PWD"
```

Then copy or symlink `$SKILL_SRC` into the coding agent's skill directory, such
as `~/.claude/skills/digital-health-create-your-own-ambient-agent` or
`~/.codex/skills/digital-health-create-your-own-ambient-agent`, and restart the agent session.
